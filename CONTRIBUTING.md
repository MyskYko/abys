# Contributing to abys

Thanks for your interest in abys! This project is early and evolving, so small, focused contributions are preferred.

## How to contribute

1. Fork the repo and create a feature branch.
2. Make changes with tests where it makes sense.
3. Run the build and tests locally.
4. Open a PR with a clear description of the change and rationale.

## Development

- CMake build is the default.
- Keep headers in `include/` and implementation in `src/`.
- Small SystemVerilog fixtures live under `tests/fixtures/`.

## Style

- Use `clang-format` (config in `.clang-format`).
- Keep code clear and avoid introducing large dependencies.

## Code of Conduct

Be respectful and constructive. We follow the MIT license and common open-source norms.
